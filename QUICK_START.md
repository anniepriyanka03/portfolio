# ⚡ Quick Start Guide

Get your portfolio running in 30 seconds!

---

## 🎯 Three Easy Steps

### Step 1: Open the Website
Simply open `index.html` in your browser by double-clicking it.

### Step 2: View Your Portfolio
You should see your complete portfolio with all 7 sections:
- Personal Details
- Education
- Technical Skills
- Projects
- Coding Platforms
- Achievements
- Contact

### Step 3: Customize It
Edit `index.html` to update your information, or modify `css/style.css` for styling changes.

---

## 🚀 Local Development Server (Optional)

For better development experience, use a local server:

### Python
```bash
python -m http.server 8000
```
Visit: `http://localhost:8000`

### Node.js
```bash
npx http-server
```

### VS Code
1. Install "Live Server" extension
2. Right-click `index.html` → "Open with Live Server"

## 📝 Quick Customization

### Update Your Name
In `index.html` (line 45):
```html
<h1 class="hero-title">Your Name Here</h1>
```

### Update Your Email
In `index.html` (line 48):
```html
<a href="mailto:your.email@example.com">your.email@example.com</a>
```

### Change Colors
In `css/style.css` (line 8-20):
```css
--accent-color: #YOUR_COLOR;
--accent-success: #YOUR_COLOR;
```

### Update Projects
In `index.html` (line 185+):
Edit each project card with your project information.

## 📤 Deploy Your Website

### FREE Option 1: GitHub Pages
1. Create GitHub account & repository
2. Push your files to GitHub
3. Enable Pages in repository settings
4. Your site is live at: `github.com/yourusername/portfolio`

### FREE Option 2: Netlify
1. Go to `netlify.com`
2. Connect your GitHub repository
3. Deploy automatically with every push

### FREE Option 3: Vercel
1. Go to `vercel.com`
2. Import your project
3. One-click deployment

## 🐛 Troubleshooting

### Website looks broken
- Clear browser cache (Ctrl+Shift+Delete)
- Try a different browser
- Check console for errors (F12)

### Images not showing
- Place images in `assets/images/` folder
- Update image paths in HTML

### Styles look wrong
- Make sure `css/style.css` is in the `css/` folder
- Reload page (Ctrl+R or Cmd+R)
